**Wordmark (D-43):**
- Family: `--font-plex-serif`
- Weight: 700
- Size: `22px` (header) / `28px` (login + public home prominent placement)
- Letter spacing: `-0.04em`
- Line height: 1
- Color: `text-hey-fg-1` (`#FFFFFF`)
- Glyph string: `data,` (lowercase + trailing comma — locked recommendation; planner may swap to `datatón,` if word balance demands)
- The trailing comma is part of the mark, not punctuation. Render as a single `<span>` so it never wraps.

**Tailwind utility shorthands** (defined in `globals.css` `@layer components`):
- `.hey-amount` → `font-family: var(--font-serif); font-variant-numeric: tabular-nums; letter-spacing: -0.02em;`

### Currency formatting

All MXN amounts pass through:
```ts
new Intl.NumberFormat("es-MX", { style: "currency", currency: "MXN", minimumFractionDigits: 2 }).format(n)
// → "$1,000.00", "$12,450.30"
```
Apply `.hey-amount` class to the rendering element so numerals use Plex Serif tabular-nums.

### Date formatting

| Distance | Output | API |
|----------|--------|-----|
| Today | "Hoy HH:mm" | `Intl.DateTimeFormat("es-MX", { hour: "numeric", minute: "numeric" })` prefixed with "Hoy " |
| Yesterday | "Ayer" | hand-rolled diff + literal |
| ≤ 30 days | "23 abr" | `Intl.DateTimeFormat("es-MX", { day: "numeric", month: "short" })` |
| Older | "23 abr 2025" | add `year: "numeric"` |

## Motion / Animation Contract

All durations in ms; all easings from the standard token unless noted.

### Easings

| Token | Value | Source |
|-------|-------|--------|
| `--hey-ease-standard` | `cubic-bezier(0.32, 0.72, 0, 1)` | Bundle's `Slide` wrapper (lines 53–60 of VariantAv2.jsx) |

### Animation Catalog

| Name | Engine | Duration | Easing | Trigger | Notes |
|------|--------|----------|--------|---------|-------|
| HAVI ring rotation | CSS `@keyframes hey-havi-spin` | `8s` linear infinite | linear | Mount, while visible | Disabled under `prefers-reduced-motion: reduce` |
| Typing dots pulse | CSS `@keyframes hey-typing-dot` | `1.2s` infinite per dot, stagger `0.15s` × index | ease-in-out | TypingIndicator visible | Disabled under reduced-motion (drop opacity steady at 0.5) |
| `fadeInUp` (chat message entry) | motion `motion.div` `initial={{ opacity: 0, y: 8 }} animate={{ opacity: 1, y: 0 }}` | `280ms` | `--hey-ease-standard` | New message append | Stagger `40ms` per index when batching multiple |
| `slide-from-right` (sub-route entry) | motion `motion.div` via `<PageTransition from="right">` | `320ms` | `--hey-ease-standard` | Sub-route page mount | `initial={{ x: "100%", opacity: 0.4 }} animate={{ x: 0, opacity: 1 }}` |
| `slide-from-bottom` (agent modal entry) | motion `motion.div` via `<PageTransition from="bottom">` | `320ms` | `--hey-ease-standard` | `/app/agente` mount | `initial={{ y: "100%", opacity: 0.4 }} animate={{ y: 0, opacity: 1 }}` |
| `growBar` (bar chart fill) | motion `motion.div` `scaleX 0→1 originLeft` | `600ms` | `--hey-ease-standard` | BarChart segment in view | Stagger `80ms` per segment |
| `scaleIn` (success check) | motion `motion.div` `scale 0.6→1, opacity 0→1` | `400ms` | `--hey-ease-standard` | TransferConfirm mount | One-shot |
| `connecting-spin` (Sofía connect ring) | CSS `@keyframes spin` | `1s` linear infinite | linear | `connecting` step | Stops when step transitions to `connected` (1800ms) |
| Suggestion chip / send button color transition | CSS `transition: background 200ms ease` | `200ms` | ease | `input.trim()` flips truthy/falsy | Skip transform; opacity-only |

### Reduced motion

Wrap chat home + sub-routes in:
```tsx
<MotionConfig reducedMotion="user">{children}</MotionConfig>
```
Place inside `app/app/template.tsx` (NOT the layout — see Pitfall 6). Globally: motion respects `prefers-reduced-motion` and disables transform/layout animations while preserving opacity. CSS keyframes (HAVI ring, typing dots) honor `@media (prefers-reduced-motion: reduce)` directly.

### Authenticator flow

The Authenticator's own internal transitions (form swap, error fade) are not customized in Phase 03.1.

## Component Inventory

All components live under `app/_components/` (private — excluded from routing). All are `"use client"` unless explicitly noted.

### `<HaviRing>`
Pure CSS conic-gradient ring with rotating animation. React wrapper only exposes size + className.

```ts
type Props = {
  size?: number;     // px, default 56
  className?: string;
  animated?: boolean; // default true; false freezes rotation (used in static decorations)
};
```
- States: default (rotating), reduced-motion (static, gradient still visible).
- `aria-hidden="true"` (decorative only — never the only signal of a label).

### `<Wordmark>`
Renders the `data,` mark with Plex Serif Bold + tracking lock.

```ts
type Props = {
  size?: "sm" | "lg"; // sm=22px (header), lg=28px (login + public home)
  as?: "span" | "h1"; // default span
  className?: string;
};
```
- Always wraps content in a single non-breaking element (`<span>` or `<h1>`); never lets the comma wrap.

### `<AppHeader>`
Per-page header. **Lives in each page**, NOT in `app/app/layout.tsx` (Pitfall 6).

```ts
type Props = {
  variant: "havi" | "back";  // havi=chat home (HaviRing+label+user avatar), back=sub-routes
  title?: string;            // for variant="back"
  backHref?: string;         // default router.back()
  closeButton?: boolean;     // for /app/agente — uses X icon instead of chevron-left
  rightSlot?: ReactNode;     // optional custom action (e.g. SignOutButton in agent screen)
};
```
- Layout: `flex h-14 items-center px-5` with `border-b border-hey-divider` on `back` variant only.
- HAVI variant left: `<HaviRing size={28}>` + label "HAVI" (`text-hey-fg-2 text-[13px] font-medium tracking-wide`).
- HAVI variant right: 32×32 user avatar circle (`bg-hey-surface-2`) with `user` icon, click → opens SignOutButton menu (or directly signs out — planner decides).
- Back variant left: chevron-left button (`size-8`, padding 6px) + serif title.
- Back variant right: small HaviRing 22px (sub-routes only) for brand continuity.

### `<Composer>`
Sticky bottom textarea + send button.

```ts
type Props = {
  value: string;
  onChange: (next: string) => void;
  onSubmit: (text: string) => void;
  placeholder?: string;       // default "Pregúntale a HAVI..."
  disabled?: boolean;
  autoFocus?: boolean;
};
```
- Wrapper: `bg-hey-surface-2/95 border border-hey-outline rounded-full pl-5 pr-1.5 py-1.5 flex items-center gap-2`.
- `<textarea>` auto-resize 1–3 lines (max-height = `3 × line-height`); `resize-none`, `bg-transparent`, `outline-none`.
- Send button: 38×38 circle. Disabled (empty trim): `bg-transparent border border-hey-outline`, send icon `text-hey-fg-2`. Enabled: `bg-hey-blue`, white icon.
- Keyboard: `Enter` submits, `Shift+Enter` newline. `Esc` blurs.
- ARIA: textarea `aria-label="Mensaje para HAVI"`, button `aria-label="Enviar mensaje"`.

### `<TypingIndicator>`
Three pulsing dots inside a HAVI-style bubble.

```ts
type Props = { ariaLabel?: string }; // default "HAVI está pensando…"
```
- `role="status"` + `aria-live="polite"`.
- Bubble: `bg-hey-surface-2 rounded-[18px] rounded-bl-[6px] px-4 py-3 inline-flex gap-1.5 self-start`.
- Three `<span class="size-1.5 rounded-full bg-white/50">` with `animation: hey-typing-dot 1.2s {i*0.15s} infinite ease-in-out`.

### `<ChatMessage>`
Discriminated-union renderer. Receives one `ChatMessage` from `app/_data/types.ts` and renders the right card.

```ts
type Props = {
  msg: ChatMessage;
  onNavigate: (href: string) => void; // router.push
  onAgent: () => void;                // router.push("/app/agente")
};
```
- Internal switch on `msg.kind`:
  - `intro` → serif greeting, 28px / weight 500 / `tracking-tight`. `aria-label="Saludo HAVI"`.
  - `user-text` → user bubble (right-aligned, `bg-hey-blue`, white text).
  - `havi-text` → HAVI bubble (left-aligned, `bg-hey-surface-2`).
  - `snapshot` → renders `<SnapshotCard>` (full-width card, gradient bg `linear-gradient(180deg, #131313 0%, #0a0a0a 100%)`).
  - `spend-actions` → text card + 2×2 grid of `<BigActionButton>`.
  - `screen-action` → row card with icon chip + title + subtitle + chevron.
  - `transfer-card` → renders `<TransferCard>`.
  - `alert-card` → renders `<AlertCard>` (variants below).
  - `fallback` → text + "Hablar con agente" primary CTA + "Reintentar" secondary.
- Each branch wraps content in `motion.div` with the `fadeInUp` preset (see Animation Catalog).
