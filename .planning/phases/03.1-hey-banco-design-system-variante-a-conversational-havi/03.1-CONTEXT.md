## Phase Boundary

Aplicar el design system 'hey, banco' (referencia en `.planning/design/hey-banco-design-system/`) al scaffold actual de datathon-2026 y construir la **Variante A "Conversacional"** profundizada (variant del bundle `project/v2/VariantAv2.jsx`) sobre la ruta protegida `/app`, reemplazando el placeholder de Phase 3.

Lo que se entrega:
1. **Design system foundation**: tokens de paleta dark + neon (50% intensity) en `globals.css`, IBM Plex Sans + Serif via `next/font/google` reemplazando Geist en `app/layout.tsx`, set de iconos hey-banco copiados a `public/icons/`, **HAVI ring** (rainbow gradient rotating, ~8s/cycle) como componente reusable.
2. **Authenticator restyle**: tema custom para `/login` (paleta dark + IBM Plex + accent #3478F6 vía el `theme` prop de `@aws-amplify/ui-react` v6) — corrige el contraste jarring entre /login default-Amplify y /app dark hey-banco.
3. **Conversational `/app`**: page principal es chat con HAVI (composer fijo abajo, mensajes arriba, snapshot card como primer mensaje del bot con balance + spent + sparkline). Pattern-match mock dispara cards de acción.
4. **Sub-pantallas via Next.js routes**: `/app/grafica/[categoria]`, `/app/movimientos`, `/app/comparativa`, `/app/agente` — cada una con animations slide-from-right (sub-screens) o slide-from-bottom (agent modal) usando framer-motion + back-arrow header.
5. **Action cards system**: alert (rojo), success (verde), warning (ámbar) — icono en círculo + título serif + cuerpo + CTAs.
6. **Handoff humano flow**: detection del fallback ("no entendí"), pantalla "Conectando…" con spinner, chat con agente mock (Sofía Martínez, online indicator).
7. **Persistencia localStorage**: mensajes del chat sobreviven refresh con version key + clear-on-mismatch.
8. **Greeting personalizado**: "hey, {firstName}" — first token del Cognito `name` attribute (ya cargado en Phase 3 via `fetchUserAttributes()`).

**No incluido en esta phase (límites firmes):**
- Backend integration (Cognito custom attributes para balance/spent, API routes, real persistence) — TODO mock data en client-side
- Voice mode (waveform, dictado) — usuario explícitamente dijo "modo voz no"
- Variante B Focus Mode — rejected en favor de Variante A
- Real LLM API (Anthropic / OpenAI) — pattern-match mock con typing delay simulando latencia. La integración con LLM real queda como follow-up post-datatón.
- `<Authenticator>` formFields custom (campos extra en sign-up) — solo theme visual, los flows de Phase 2 D-25 quedan intactos
- Mobile native (PWA, push notifications) — web-first
- i18n / multi-idioma — hard-locked a es-MX (parte del adoption "hey, banco")
- Accessibility audit completo — basic ARIA pero no full WCAG 2.1 AA validation
- Tests automatizados — sigue el "no test runner" de Phase 1-3 (manual + static validation)

### Brand Identity & Visual Foundation (D-42..D-45)

- **D-42:** **Adopción visual completa de hey, banco con copy es-MX** — paleta dark + neon (50% intensity), IBM Plex Sans + Serif typography, conversational tone con tú informal, copy en español mexicano (`tus dudas`, `nuestros productos`, `Pregúntale a HAVI`), formato de moneda MXN ($1,000.00 con coma thousands). Greeting: "hey, {firstName}" — del Cognito `name` attribute (D-50). NO usar el wordmark literal "hey," de Banregio (D-43) — solo la estética y el voice/tone.

- **D-43:** **Custom wordmark del proyecto** (NO `assets/wordmark.svg` del bundle, que es propiedad de Banregio). Crear wordmark tipográfico en IBM Plex Serif Bold — sugerencia: `data,` (eco visual del `hey,` con coma) o `datatón` lowercase. Wordmark vive en headers de `/`, `/login`, y top de `/app`. Claude's discretion en el shape exacto — el planner / executor decide entre las opciones tipográficas. ~24px alto, blanco sobre dark.

- **D-44:** **Custom Authenticator theme** para `/login` — usar el `theme` prop de `@aws-amplify/ui-react` v6 con:
  - `--amplify-colors-background-primary: #000000`
  - `--amplify-colors-background-secondary: #1A1A1A`
  - `--amplify-colors-font-primary: #FFFFFF`
  - `--amplify-colors-font-interactive: #3478F6`
  - `--amplify-colors-brand-primary-*: #3478F6` family
  - `--amplify-fonts-default-static: "IBM Plex Sans", system-ui, sans-serif`
  - Border-radius pill on inputs/buttons (matches design system corner radii)
  Phase 2 D-24 explicitamente dejó "theming custom = polish phase futura" — Phase 03.1 ES esa polish phase. Los flows D-29 de Phase 2 (sign-up, sign-in, sign-out, reset password, persistencia) DEBEN seguir verificables en manual flow al final de 03.1 para evitar regression.

- **D-45:** **Replace Geist completamente con IBM Plex** — `app/layout.tsx` carga `IBM_Plex_Sans` (weights 400, 500, 700) + `IBM_Plex_Serif` (weights 400, 700) via `next/font/google`. Tailwind 4 `@theme` directive maps `font-sans → var(--font-plex-sans)` y `font-serif → var(--font-plex-serif)`. Esto reemplaza el deliverable de Phase 1 (Geist) — accepted given Full adoption.

### Architecture: Routing & State (D-46..D-47)

- **D-46:** **Next.js routes URL-driven** para sub-screens del flow conversational:
  - `app/app/page.tsx` — chat home (replaces Phase 3 placeholder)
  - `app/app/grafica/[categoria]/page.tsx` — category breakdown (bar chart con animation `growBar`)
  - `app/app/movimientos/page.tsx` — full transaction list
  - `app/app/comparativa/page.tsx` — month vs prior month comparison
  - `app/app/agente/page.tsx` — human agent handoff (chat with mock agent Sofía Martínez)
  - `app/app/layout.tsx` — REUSE (Phase 3 mounted `<AuthGuard>` aquí — Phase 03.1 NO lo toca)
  Cada sub-screen tiene back-button del header que `router.back()` al chat. Browser back nativo también funciona. URLs shareables ("Mira mi gasto en Comida": `/app/grafica/comida`). Suspense boundaries en cada page para data fetching futuro.

- **D-47:** **localStorage persistence con version key** para los mensajes del chat HAVI:
  - Key: `havi-chat-v1` (string con `v1` parte del key — bump a `v2` invalida estado viejo)
  - Shape: `{ version: 1, messages: ChatMessage[], updatedAt: ISO }`
  - Read en mount (con try/catch + JSON.parse fallback a default state)
  - Write debounced (~250ms) cuando cambia messages array
  - Al refresh — restore. Al sub-screen y back — mantiene (mismo mount o re-hydratado). Al sign-out — clear (suscribir al `Hub.listen('auth', ...)` o llamar clear desde el SignOutButton).

### HAVI Brain & Mock Data (D-48..D-50)

- **D-48:** **Pattern-match mock con typing delay** — HAVI no llama LLM real. Implementación:
  - Tabla de patterns en `app/_data/patterns.ts`: regex/string match → response generator
  - Trigger words principales: `transferir|enviar`, `gast(o|é|amos)`, `pagar tarjeta`, `meta|ahorro`, `comparar|comparativa`, `movimientos|histor(ial|ia)`
  - Inputs no-match → fallback "No entendí — ¿quieres hablar con un agente?" con CTA al handoff humano
  - Cada response genera 0-N cards (action card, alert/success/warning, snapshot, etc.)
  - **Typing delay:** 800-1500ms randomized antes de mostrar la respuesta + indicator "HAVI está pensando…" (3 dots pulse). Simula latencia LLM sin costo.
  - Reabrir como follow-up post-datatón si quieres LLM real (instrucciones en deferred).

- **D-49:** **Mocks tipados en `app/_data/mock.ts`** — módulo dedicado que exporta:

- **D-50:** **Greeting "hey, {firstName}" del Cognito name attribute** — usar la pipe ya cargada en Phase 3 D-39:
  - `fetchUserAttributes()` retorna `{ email, name, sub, ... }`
  - Extraer `name` (que es el `fullname` declarado en Phase 2 D-18, OIDC standard claim)
  - First token: `firstName = name.split(" ")[0]` — "Fernando Mauro" → "Fernando"
  - Fallback chain: `firstName ?? email.split("@")[0] ?? "amigo"` — graceful degradation si attributes fail
  - Cache result en localStorage o React state para no re-fetch en cada nav

### Animations & Transitions (D-51)

- **D-51:** **framer-motion (`bun add framer-motion`, ~60kb gzip)** para todas las animations:
  - Route transitions (slide-from-right entry de sub-screens) usando `<AnimatePresence>` + `motion.div` con `initial`, `animate`, `exit` variants
  - Modal slide-from-bottom para agent handoff
  - `fadeInUp` para nuevos mensajes en chat (`stagger` para múltiples cards)
  - `growBar` para barras de gráfica (`scaleX` from 0 to 1, ~400ms ease-out)
  - `scaleIn` para success check icons
  - HAVI ring rotación continua (8s/cycle) — puede ser CSS-only (`@keyframes rotate`) si framer es overkill para esto
  - Typing indicator (3 dots pulse) — CSS keyframes
  - Spring physics conservadoras: NO bouncy overshoot, NO parallax (el design dice "no springy overshoot")
  - Acceptable trade-off vs CSS-only: ~60kb gzip + smoother route transitions + cleaner code than hand-rolled

## Specific Ideas

- **Variante A v2 es la fuente de verdad de implementación** — `.planning/design/hey-banco-design-system/project/v2/VariantAv2.jsx` (402 líneas) tiene la estructura completa que el planner debe portar a Next.js 16 + React 19 + Tailwind 4 + framer-motion. NO copiar el código verbatim — el JSX usa estilos inline, React 16/17 patterns, sin TypeScript. Refactor a:
  - Functional components con TypeScript types
  - Tailwind utility classes (no inline styles, salvo en `style={{ ... }}` para CSS variables custom)
  - framer-motion `motion.div` en lugar del `<Slide>` wrapper hand-rolled
  - Next.js routes en lugar del React state `view`
  - Pero la VISUAL OUTPUT debe coincidir pixel-perfect (per design bundle README: "recreate them pixel-perfectly")

- **Pattern-match dictionary structure** sugerida:
  ```typescript
  // app/_data/patterns.ts
  export type HaviPattern = {
    match: RegExp | ((input: string) => boolean);
    response: (input: string) => HaviResponse;
  };
  export const patterns: HaviPattern[] = [
    { match: /transferir|enviar a/i, response: () => ({ kind: "transfer-card", amount: null }) },
    { match: /gast(é|amos|o)/i, response: () => ({ kind: "spend-cards", linkTo: "/app/grafica/general" }) },
    ...
    { match: /.*/, response: () => ({ kind: "fallback", offerHumanAgent: true }) }, // catch-all
  ];
  ```

- **Greeting fallback chain** explícito:
  ```typescript
  const firstName = (await fetchUserAttributes()).name?.split(" ")[0]
    ?? user?.signInDetails?.loginId?.split("@")[0]
  ```
